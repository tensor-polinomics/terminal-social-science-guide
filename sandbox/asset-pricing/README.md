# A Signal-Sorted Long-Short Portfolio: an FF5 Example

This is the running example for the book *Terminal & Shell for
Social Science Researchers*. It is a small, fully simulated
empirical asset-pricing study: does an accounting signal earn
a long-short return that the five Fama-French factors do not
span (the "mispricing" alpha)?

**All data are simulated.** No real CRSP, Compustat, or
Bloomberg data are used. The raw panel is generated by a
seeded script, so the whole project rebuilds from code alone.
Treating the synthetic data exactly as we would treat licensed
data (rebuilt from code, never committed) is the teachable
moment, not the econometrics.

## The pipeline (one `make` graph)

```
00_make_data.py  seed -> data/raw/{firm_panel,factors}.csv
01_clean.py      build + lag signal -> analysis.parquet
02_portfolio.py  long-short decile sort -> portfolio.{parquet,csv}
03_figure.py     cumulative return -> output/figures/cumret.png
04_regression.R  FF5 regression -> output/tables/ff5_alpha.{tex,txt}
report.qmd       assemble table + figure -> report.html / .pdf
```

Python (00-03) runs anywhere, including the locked-down
sandbox. **R (04) and the Quarto render run on the Mac only.**

## How to reproduce

On the Mac (full build):

```bash
uv sync                          # Python env from uv.lock
Rscript scripts/bootstrap_renv.R # once: writes renv.lock
make                             # build through report.html
make check                       # re-derive hash + alpha
quarto render report.qmd --to pdf  # PDF (separate step)
```

`make` builds the graph up to the HTML report; the PDF is a
separate `quarto render` because `all` targets `report.html`.

Python half only (e.g. in a Linux sandbox without R):

```bash
uv run python scripts/00_make_data.py
uv run python scripts/01_clean.py
uv run python scripts/02_portfolio.py
uv run python scripts/03_figure.py
uv run python scripts/check.py   # skips the R-table check
```

## Locked expected result (the reproducibility invariant)

| Quantity | Value |
|---|---|
| Seed | `20260629` |
| Lambda (signal -> return) | `0.00149` |
| Panel | 300 firms x 180 months (179 after the lag) |
| Idiosyncratic vol | `0.060` monthly |
| Data content SHA-256 | `49b3a1733cf22f5339f1f56edb50bcb6ae02e8d6a029b8f894f279cdf55dbccb` |
| Long-short alpha | `0.0034` per month (4 dp), t approx 2.64 |
| Alpha, annualized | approx 4.1% per year |
| Figure | cumulative long-short return, ends approx 104% |
| Factor loadings | 4 of 5 insignificant; HML marginal (t approx -1.8) |

The canonical alpha comes from R (`04_regression.R`, default
iid SE). The in-sandbox `statsmodels` cross-check in
`check.py` reproduces the same intercept to 4 dp; the R run on
the Mac must land on the same `0.0034`. The data SHA-256 is
computed over the stored (6-dp) returns, factors, and RF, so
it is stable across machines.

## Layout

```
scripts/    numbered; 00-03 Python, 04 R, plus check.py
data/raw/   simulated raw data         (git-ignored)
data/clean/ derived analysis files     (git-ignored)
output/     figure + R table, rebuilt by make (git-ignored)
Makefile    the cross-language build graph
report.qmd  the short Quarto/LaTeX report
```

## Environment

Python 3.10+ with numpy, polars, pyarrow, matplotlib, and
statsmodels; pinned in `uv.lock`. R with fixest (regression)
plus knitr and rmarkdown (Quarto report); pinned in
`renv.lock` (created on the Mac by `scripts/bootstrap_renv.R`).
