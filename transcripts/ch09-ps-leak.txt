note: the leak demo and its mitigation. A secret passed as a
  command-line argument (--password=hunter2) shows up verbatim
  in the process arg list, which ps prints (de-quoted, as the
  kernel stores argv) to any user on the machine. A program
  that reads the key from the environment instead (readkey.py)
  shows no secret in its args. hunter2 and the sk-... string
  are fake placeholders. PIDs vary per run. `ps -o pid,args`
  prints no owner column, so nothing is masked.

The mitigation script (readkey.py):

  import os, time
  key = os.environ["API_KEY"]   # read from env, not argv
  time.sleep(30)
